# NowSDK / Build-Agent UI Page Apps — Hints

## Key Tables

| Table | Purpose |
|-------|---------|
| `sys_ui_page` | One record per entry-point page; HTML shell is auto-generated by the build tool |
| `sys_ui_script` | Stores the compiled JS bundle(s); served at `/uxasset/externals/<scope>/main.jsdbx` |
| `sys_scope` / `sys_package` | The scoped application container the app lives in |

## How It Works

NowSDK (also called "build-agent generated" or "Fluent") apps are **single-page applications (SPAs)** deployed inside a ServiceNow scoped app.

### Deployment model
- Source lives in a **local NowSDK project** (not editable in ServiceNow records)
- Deploy command: `snc ui-component deploy` (or `snc app-install`)
- On deploy the build tool:
  1. Compiles a single JS bundle → stored in `sys_ui_script`, served as `/uxasset/externals/<scope>/main.jsdbx`
  2. Auto-generates one `sys_ui_page` HTML shell per entry point
  3. All records land in the scoped app's current update set

### The HTML shell
Each `sys_ui_page` has a near-identical minimal HTML shell:
```html
<head>
  <!-- @fluent-import-html WARNING — auto-generated, do not edit directly -->
  <script type="module" src="/uxasset/externals/<scope>/main.jsdbx?uxpcb=..."></script>
</head>
<body>
  <div id="root"></div>
</body>
```
The `@fluent-import-html WARNING` comment is the marker that this file was generated by the build tool.

### Routing
- **One bundle serves all pages** in the app — every `sys_ui_page` in the scope loads the same `main.jsdbx`
- The bundle must route itself based on `window.location.pathname`
- No server-side discriminator is injected into the HTML to tell the bundle which page it's on
- If a route is missing from the client-side router, the SPA falls through to its **default route**

## Gotchas

- **Do not edit `sys_ui_page.html` directly** — it will be overwritten on the next deploy
- **Routing bugs manifest as the wrong component rendering**: if `audit_dashboard` shows KB Authoring, the most likely cause is that the `audit_dashboard` route is not registered in the bundle's client-side router
- **All pages look identical at the record level** — to understand which component a page renders you must read the bundle source, not the `sys_ui_page` record
- **Bundle cache-busting**: the `?uxpcb=` query param on the script tag is a cache-buster injected at build time; it changes on every deploy

## Diagnosing a "wrong page renders" bug

1. Confirm all `sys_ui_page` records in the scope load the same bundle path (they will)
2. Check whether the bundle's client-side router has a route registered for the affected page URL
3. Fix is always in the **NowSDK source project** — rebuild and redeploy
4. After redeploy, verify the new `?uxpcb=` value changed (confirms the bundle was updated)

## Efficient query to find all pages in a scoped app

```bash
simon query sys_ui_page \
  --query "sys_scope.scope=<scope_prefix>" \
  --fields "name,title,sys_scope,sys_id"
```
